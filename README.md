# FilmSearcher

React + TypeScript агрегатор фильмов с двумя режимами:

- **Regular** — обычный поиск, категории и жанры через TMDB;
- **AI** — свободный запрос на русском или английском и семантический поиск прямо в браузере.

## Запуск

```bash
npm install
npm run dev
```

## Как работает AI-поиск

В продакшен не требуется отдельный backend, база данных или платный AI API:

1. `movies.json` хранит данные 50 000 фильмов из TMDB.
2. `embeddings.int8.bin` хранит заранее рассчитанные 384-мерные векторы.
3. Web Worker загружает `Xenova/multilingual-e5-small` через Transformers.js.
4. В браузере векторизуется только пользовательский запрос.
5. Worker сравнивает запрос со статическим индексом и добавляет небольшой quality prior из популярности, рейтинга и числа голосов.
6. Лучшие совпадения возвращаются в виде обычных карточек.
7. Если индекс недоступен, приложение запрашивает около 60 кандидатов у TMDB и ранжирует их той же локальной моделью.

Модель закреплена на конкретной ревизии, а документы и запросы используют E5-префиксы `passage:` и `query:`. Индекс хранится в `int8`: 50 000 × 384 = 19 200 000 байт вместо примерно 76,8 МБ для `float32`.

## Пересборка каталога

Нужен TMDB API key только во время подготовки данных:

```bash
TMDB_API_KEY=your_key npm run index:collect
npm run index:embed
```

Оба этапа возобновляются с checkpoint после прерывания. Параметры можно менять:

```bash
TMDB_API_KEY=your_key npm run index:collect -- --target=1000 --year-from=2000
npm run index:embed -- --limit=1000 --batch-size=64
```

Готовые файлы записываются в `public/movie-index/` и автоматически входят в сборку Vite/Vercel.
